Sección 07 · Ejercicio práctico

Tu primer flujo completo
con IA en VS Code

Este ejercicio te lleva paso a paso por el flujo completo que usaremos en IT4W. No hace falta saber programar — solo seguir los pasos. Tiempo estimado: 20–30 minutos.

"Git no guarda en otro lado — guarda la historia dentro de tu carpeta."

1
Modo: Ask · Duración: 5 min
Explorar el proyecto
Entender qué tiene la carpeta antes de hacer cualquier cosa

Abrí VS Code en la carpeta del proyecto. Activá el modo Ask en el chat de Copilot y escribí:

¿Qué archivos tiene este proyecto y cuál es el propósito de cada carpeta?

La IA va a leer la estructura del proyecto y te va a explicar qué hay. No modifica nada, solo te informa.

💡 Si no ves el chat de Copilot, buscá el ícono de chat en la barra lateral izquierda de VS Code o usá el menú Ver → GitHub Copilot Chat.

Cuando tengas el panorama general, anotá mentalmente o en papel:

  • ¿Cuántas carpetas hay?
  • ¿Hay archivos .md? ¿Para qué sirven?
  • ¿Qué es la carpeta skills-main?
2
Modo: Plan · Duración: 5 min
Planificar la sección
Que la IA te proponga cómo abordar la tarea antes de ejecutarla

Cambiá el modo a Plan en el selector del chat y escribí:

Quiero documentar qué son los modos de trabajo (Ask, Plan, Agent) para personas no técnicas del área de UX y Comercial. ¿Cómo lo encarás?

La IA va a proponer una estructura: qué incluir, qué orden usar, qué ejemplos dar. Todavía no escribe nada, solo te muestra el camino.

Leé la propuesta y verificá:

  • ¿Tiene introducción, cuerpo y cierre?
  • ¿Menciona ejemplos prácticos?
  • ¿El tono parece accesible para no-técnicos?
Si la propuesta no te convence, podés decirle: "Agregá más ejemplos del área Comercial" o "Hacelo más corto". El Plan es tuyo para modificar.
3
Modo: Agent · Subagente: /generar-seccion · Duración: 5–8 min
Generar la sección
La IA escribe el contenido completo y lo guarda en un archivo

Cambiá el modo a Agent. Ahora vas a invocar el subagente que genera secciones:

/generar-seccion concepto: "Los modos de trabajo: Ask, Plan y Agent" archivo: contenido/mi-seccion.md

El subagente va a generar el texto y guardarlo en el archivo contenido/mi-seccion.md. Cuando termine, verás el archivo nuevo en la carpeta.

💡 Si el subagente pide confirmación para escribir el archivo, confirmá con "sí" o "ok".

Abrí el archivo generado y revisá:

  • ¿Tiene los tres modos explicados?
  • ¿El lenguaje es accesible?
  • ¿Hay al menos un ejemplo por modo?
4
Modo: Agent · Subagente: /revisar-simplicidad · Duración: 5 min
Revisar la sección
Evaluar la claridad antes de dar el texto por bueno

Con el modo Agent activo, invocá el subagente de revisión:

/revisar-simplicidad archivo: contenido/mi-seccion.md

El subagente va a evaluar el texto y darte un puntaje del 1 al 10 junto con sugerencias. Interpretá el resultado:

1–4 El texto es confuso o demasiado técnico. Hay que reescribir.
5–6 Mejorable. Incorporá las sugerencias y volvé a generar.
7–8 Bueno. Podés continuar al siguiente paso.
9–10 Excelente. Listo para diseñar.
⚠️ Si el puntaje es menor a 7, volvé al Paso 3 con las sugerencias. No avances al diseño con un texto que no está claro.
5
Git · Duración: 3–5 min
Guardar en Git
Dejar registro de tu trabajo en el historial del proyecto

Git guarda la historia de todos los cambios del proyecto. Para registrar lo que hiciste, abrí la terminal de VS Code (Ctrl + ` en Windows) y ejecutá estos comandos en orden:

git add contenido/mi-seccion.md
git commit -m "feat: seccion modos de trabajo — ejercicio de inicio"

¿Qué hace cada comando?

  • git add — dice "este archivo va en el próximo guardado"
  • git commit — hace el guardado con una descripción de lo que hiciste
💡 Si VS Code te pregunta quién sos la primera vez que hacés un commit, escribí tu nombre y email cuando te lo pida. Es solo para identificar los cambios.

Cuando el comando se ejecute sin errores, tu trabajo quedó registrado. Podés verificarlo con:

git log --oneline -5

Vas a ver el listado de los últimos commits, incluyendo el que acabás de hacer.

🎉 ¡Completaste el ejercicio!

Usaste los tres modos (Ask, Plan y Agent), dos subagentes, evaluaste la calidad del texto y guardaste tu trabajo en Git. Eso es exactamente cómo trabaja el equipo IT4W.
