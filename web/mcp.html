Sección 05

MCP: el puente hacia el mundo exterior

La IA en VS Code trabaja bien con lo que está dentro de tu proyecto. Pero ¿qué pasa cuando necesitás que también use información de Figma, GitHub, o de otras herramientas que usás todos los días? Ahí entra MCP.

"MCP es el teléfono que le permite a la IA hablar con herramientas fuera de VS Code."

MCP son las siglas de Model Context Protocol. Es un estándar que le permite a la IA conectarse con herramientas externas y leer o interactuar con ellas directamente — sin que vos tengas que copiar y pegar nada.

Ejemplos en IT4W

🎨
MCP de Figma
Figma + IA
La IA puede leer directamente tu archivo de Figma: componentes, estilos, frames. Podés pedirle que genere documentación de diseño, checklists de accesibilidad, o specs para developers — sin salir de VS Code.
Ejemplo "Revisá el Design System en Figma y listá los componentes sin estado :disabled documentado."
🐙
MCP de GitHub
GitHub + IA
La IA puede consultar issues, pull requests y el historial del repositorio directamente. Útil para generar reportes de avance, checklists de review o comentarios automáticos.
Ejemplo "Listá todos los issues abiertos del sprint y generá un resumen para la reunión de cierre."
⚠️ MCP solo funciona en modo Agent. Si estás en modo Ask o Plan, la IA no puede conectarse con herramientas externas.
⚠️ Solo instalá MCPs oficiales. Hay MCPs de terceros que pueden tener acceso a tus datos. Usá únicamente los que están verificados por Anthropic, GitHub u otras fuentes confiables.
💡 MCP se configura una sola vez por proyecto, en el archivo settings.json de VS Code o en la configuración de Copilot. Consultá con tu líder de área para acceder a los MCPs habilitados en IT4W.
