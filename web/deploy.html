Sección 08

Deploy y publicación:
tu trabajo, visible para todos

⏱️ Duración estimada: 20 min
📖 Concepto

¿Qué es el deploy?

Deploy significa publicar. Es el acto de tomar lo que tenés en tu computadora y dejarlo accesible en internet para que cualquier persona pueda verlo con un link.

"Tu compu es el taller. GitHub es la bodega. Render es la vidriera. Vos trabajás en el taller y decidís cuándo poner algo en la vidriera."

Hoy en IT4W usamos tres piezas que se conectan en cadena:

💻 Tu compu — donde editás y guardás cambios
↓ git push
GitHubGuarda el historial de versiones en la nube
↓ automático
RenderDetecta el cambio y publica la versión nueva
↓
🌐 URL pública — accesible para cualquier persona
💡 El flujo del día a día

Desarrollo vs. producción

Estas palabras suenan técnicas pero el concepto es simple:

🔧 Desarrollo (tu compu)
Donde hacés los cambios. Nadie más lo ve. Podés romper cosas, probar, deshacer. Es tu espacio de trabajo.
🌐 Producción (la URL pública)
Lo que ven todos. Solo llega acá lo que vos decidís publicar con un push. Es la versión "oficial".

El flujo completo de cada cambio que hacés:

PasoQué hacésQué pasa
1. EditásCambiás un archivo en VS CodeEl cambio existe solo en tu compu
2. Commitgit commit -m "descripción"Git guarda una foto del estado actual
3. Pushgit pushSube esa foto a GitHub
4. DeployRender lo detecta soloLa URL pública se actualiza en ~1 min
💡 Vos solo hacés los pasos 1, 2 y 3. El 4 pasa solo — Render vigila GitHub las 24 horas.
🛠️ Ejercicio — Publicá tu primer cambio

Editá algo, guardalo y velo en vivo

Vas a hacer un cambio pequeño en la guía, publicarlo y ver el resultado en la URL pública en tiempo real.

1
Explorador de VS Code
Hacé un cambio simple en la guía
Algo pequeño que puedas reconocer cuando lo veas publicado

Abrí cualquier archivo HTML de la carpeta web/ — por ejemplo web/index.html. Buscá algún texto visible (un título, una descripción) y cambiá una palabra. Algo que puedas identificar fácilmente cuando lo veas en el browser.

Ejemplo: En index.html, buscá la frase "Para UX, Comercial, QA y toda el área." y cambiala por algo como "Para todo el equipo de IT4W."

Guardá el archivo con Ctrl + S (Windows) o Cmd + S (Mac).

💡 No rompas nada importante — elegí un texto de poco impacto. Si algo sale mal, siempre podés deshacer con Ctrl + Z.
2
Panel de chat de VS Code — modo Agent
Hacé el commit — guardá la foto del cambio
Registrar el cambio en el historial de Git

Un commit es como sacarle una foto al estado actual de tu proyecto. Incluye todos los archivos que modificaste y un mensaje que describe qué cambiaste.

Pedile al agente que lo haga por vos:

💬 Copiá esto en el chat — modo Agent 
Hacé un commit con todos los archivos que modifiqué. El mensaje del commit tiene que describir brevemente qué cambié.

El agente va a detectar qué archivos cambiaron, escribir un mensaje descriptivo y hacer el commit por vos.

Vas a ver en el chat algo como: git commit -m "Actualiza texto de descripción en index.html" y luego una confirmación.

3
Panel de chat de VS Code — modo Agent
Hacé el push — subí el cambio a GitHub
Enviar el commit a la nube para que Render lo detecte

El commit existe en tu compu, pero todavía no en GitHub. El push lo sube:

💬 Copiá esto en el chat — modo Agent 
Hacé un push para subir el último commit a GitHub.

El agente ejecuta git push. Cuando termina, el cambio ya está en GitHub y Render empieza a procesar el deploy automáticamente.

⚠️ Si te pide usuario y contraseña de GitHub, ingresalos. Si ya los tenés guardados, pasa directo.
4
Navegador
Verificá el cambio en la URL pública
Ver tu cambio publicado en tiempo real

Esperá 1-2 minutos y abrí la URL de la guía en el navegador:

https://guia-ia-it4w.onrender.com

¿Ves el texto que cambiaste? Si sí — ¡el flujo completo funcionó! Editaste localmente, commiteaste, pusheaste a GitHub, Render lo detectó y publicó.

✅ Si ves el cambio: Ya sabés hacer deploy. Este es el flujo que vas a repetir cada vez que quieras publicar algo nuevo.
🕐 Si no ves el cambio todavía: Render puede tardar hasta 2-3 minutos. Actualizá la página con Ctrl + Shift + R (fuerza la recarga sin caché).

✅ Checkpoint

¿Qué aprendiste en esta sección?

  • 🌐 Deploy es publicar tu trabajo en una URL accesible para todos.
  • 🔧 Desarrollo es tu compu — donde trabajás sin que nadie vea.
  • 📦 Commit es guardar una foto del estado de tu proyecto con Git.
  • ⬆️ Push es subir esa foto a GitHub.
  • 🤖 Render detecta cada push y actualiza la URL pública automáticamente.
  • 💬 El agente puede hacer el commit y el push por vos — solo pedíselo.
