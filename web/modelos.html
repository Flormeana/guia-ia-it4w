🗺️ Las herramientas disponibles

No son competencia — son etapas

IT4W tiene acceso a varias IAs. Cada una tiene un momento ideal en el flujo de trabajo.

HerramientaDónde vivePara qué sirveCuándo usarla
M365 Copilot Teams, Word, Outlook Pensar, resumir, ordenar ideas, crear esquemas y borradores Antes de abrir VS Code — para estructurar qué vas a hacer
Copilot en VS Code Chat de VS Code Analizar archivos, razonar, estructura técnica Cuando necesitás precisión y acceso al repositorio
Claude en VS Code Chat de VS Code Redactar, documentar, comunicar en lenguaje humano Cuando necesitás tono natural y textos para personas reales
Copilot CLI Terminal, fuera del editor Delegar tareas largas de forma autónoma, sin cargar el editor Cuando VS Code va lento o la tarea es muy pesada
🧮 ¿Qué versión de Claude elegir?

Dentro del selector de VS Code vas a ver varios modelos de Claude. Cada uno tiene un costo distinto en tokens:

ModeloCosto en tokensCuándo usarlo
Claude Sonnet (el más reciente)✅ 1×Día a día — redacción, análisis, documentación. La opción ideal para la mayoría de los casos.
Claude Opus (el más reciente)⚠️ 3×Tareas muy complejas que requieren el máximo razonamiento. Usalo cuando Sonnet no alcanza.

"Más reciente" = el número de versión más alto que aparece en el selector. Si ves Sonnet 3.5 y Sonnet 3.7, elegí el 3.7. Si ves Sonnet 4, ese es.

💡 Tip de tokens
Para borradores largos sin código, usá M365 — reservá el chat de VS Code para trabajar directamente sobre los archivos.
💡 Tip — diseño de prompts
Si tenés una idea difusa, pedile a M365 que te ayude a formularla. Después pegás ese prompt en Claude dentro de VS Code.

2
Copilot CLI — requiere instalación previa
Conocé el CLI y cómo instalarlo
Entendés qué es, cómo se configura y cuándo vale la pena
⚠️ Copilot CLI no viene instalado por defecto. Es una herramienta separada que hay que configurar. Si no la tenés, el comando de más abajo no va a funcionar — eso es normal y esperado.

¿Qué es? Es Copilot funcionando en la terminal — una ventana de texto, sin interfaz visual. Escribís un comando, él ejecuta la tarea y devuelve el resultado. Como no carga el editor gráfico, usa muy poca RAM.

Cómo abrir la terminal:

Dentro de VS Code:
  Windows: Ctrl + `  (acento grave — arriba a la izquierda, junto al 1)
  Mac: Cmd + `
  Por menú: Terminal → Nueva terminal
  ✓ Si ya tenés el proyecto abierto en VS Code, la terminal abre automáticamente en esa carpeta.

Fuera de VS Code (terminal del sistema):
  Windows: Win + R → escribí powershell → Enter
  O: Win + X → "Terminal Windows"
  Mac: Cmd + Espacio → escribí terminal → Enter
  ⚠ La terminal abre en una ubicación genérica — tenés que navegar hasta tu proyecto.

Si usás la terminal fuera de VS Code, escribí este comando para ir a tu proyecto (reemplazá con tu ruta real):

cd "C:\Users\TU-USUARIO\Desktop\mi-proyecto-ia"

cd significa "cambiar directorio" — es como hacer doble click en una carpeta. Una vez que lo ejecutás, todos los comandos que escribas van a afectar solo a los archivos de esa carpeta.

Cómo instalarlo — paso a paso:

① Instalá GitHub CLI

Windows — en la terminal escribí:

winget install --id GitHub.cli

Mac — en la terminal escribí:

brew install gh

¿Preferís instalador con clicks? Descargalo desde cli.github.com

② Autenticarte con tu cuenta de GitHub

gh auth login

Elegí GitHub.com y después Login with a web browser. Va a abrirse el navegador para confirmar tu cuenta.

③ Activá el Copilot CLI

Corré este comando. La primera vez te pregunta "Install GitHub Copilot CLI? (y/N)" — escribí y y apretá Enter:

gh copilot --version

Cuando termina, muestra el número de versión. Eso significa que quedó instalado.

Una vez instalado, se usa así:

⌨️ Ejemplo de uso 
gh copilot -p "lo que querés hacer"

Sin chat, sin historial visual. El resultado aparece directo en la terminal.

Lo que aprendiste en esta sección

  • 🗺️ M365, VS Code y CLI no compiten — cada una tiene un momento en el flujo.
  • 📋 M365 Copilot es para pensar y estructurar antes de ejecutar.
  • 🤖 Copilot en VS Code: análisis y razonamiento técnico.
  • ✨ Claude en VS Code: redacción y comunicación en lenguaje humano. Sonnet consume menos tokens.
  • ⚡ Copilot CLI: autónomo y liviano — para cuando el editor va lento o la tarea es pesada.
  • 🔄 Cambiás de modelo con el selector abajo del cuadro de texto.
