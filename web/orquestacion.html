Sección 06

Orquestación: un pedido,
todo el equipo trabajando

La orquestación es cuando el agente principal coordina a múltiples subagentes para completar una tarea compleja, encadenando sus resultados de forma automática.

"El agente principal es el director de orquesta: vos le decís qué querés lograr, y él decide qué especialista llamar y en qué orden."
Es como encargar una propuesta comercial: vos le decís al gerente de cuenta "necesito la propuesta para el lunes", y él coordina al redactor, al diseñador y al financial — sin que vos tengas que hablar con cada uno.

El mismo trabajo, antes y después

❌ Antes de la orquestación
Para generar una sección de la guía, necesitabas hacer tres pedidos por separado: uno para generar, otro para revisar, otro para diseñar. Si olvidabas el orden, el resultado era inconsistente.
✅ Con orquestación
Hacés un solo pedido: "generá la sección de KPIs para el área de Analytics". El agente principal llama a /generar-seccion, luego a /revisar-simplicidad, evalúa el score y recién ahí llama a /diseñar-seccion.

El flujo completo

🎯 Pedido: "Generá la sección sobre KPIs de Analytics"
↓ el agente principal lee las instrucciones del proyecto
Paso 1 · /generar-seccion Genera el contenido de la sección con el concepto pedido
↓
Paso 2 · /revisar-simplicidad Evalúa claridad, puntaje del 1 al 10
↓
¿Score ≥ 7? → Continuar   |   Score < 7 → Volver a Paso 1
↓
Paso 3 · /diseñar-seccion Genera el HTML visual con el contenido aprobado
↓
✅ Sección lista · Contenido + diseño + consistencia garantizada

Ejemplo completo: equipo UX

Sofía de UX necesita documentar tres pantallas del nuevo flujo de onboarding.

  1. Sofía abre el chat de Copilot en modo Agent
  2. Escribe: "Documentá las tres pantallas del flujo de onboarding: bienvenida, perfil y confirmación"
  3. El agente lee las instrucciones del proyecto y detecta que hay que usar el subagente /generar-seccion tres veces
  4. Para cada pantalla: genera → revisa → aprueba → diseña
  5. Al final, las tres secciones están generadas, revisadas y diseñadas en el mismo formato
💡 La orquestación funciona mejor cuando las instrucciones del proyecto están bien escritas en el archivo .github/copilot-instructions.md. Ahí es donde el agente principal sabe qué subagentes usar y cuándo.
