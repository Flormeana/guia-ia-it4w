Sección 06

Git: el historial de tu proyecto

Cada vez que trabajás con la IA en VS Code, producís cosas: archivos, documentos, cambios. Git es el sistema que guarda la historia de todo eso — para que puedas volver atrás, compartir tu trabajo y no perder nada.

"Git no guarda en otro lado. Guarda la historia dentro de tu carpeta."

¿Qué es Git?

Git es un sistema de control de versiones. Eso significa que registra cada cambio que hacés — quién lo hizo, cuándo y qué cambió.

No es una nube. No es un servicio externo. Git es un sistema que vive dentro de tu propia carpeta de proyecto. Cuando hacés git init, creás una carpeta oculta llamada .git que empieza a guardar todo el historial.

Los 3 comandos que necesitás

ComandoQué haceAnalogía
git init Inicializa Git en la carpeta actual Abrís un cuaderno nuevo para el proyecto
git add . Marca todos los archivos para guardar Seleccionás las fotos que querés revelar
git commit -m "mensaje" Guarda los cambios con una descripción Revelás las fotos y las ponés en el álbum
git log Muestra el historial de commits Abrís el álbum y ves todas las fotos

El flujo básico

git init
git add .
git commit -m "Primera versión de mi proyecto"

Eso es todo. Tres comandos. Tu proyecto ya tiene historial.

git log --oneline

Esto muestra los commits en una línea cada uno — el más reciente arriba.

¿Por qué usarlo en IT4W?

  • No perdés nada. Si algo sale mal, podés volver a cualquier punto anterior de tu proyecto.
  • Podés compartir. Git es la base de GitHub y SharePoint — los sistemas que usamos para colaborar en IT4W.

Git ≠ GitHub

Git es el sistema que corre en tu máquina. GitHub es una plataforma online donde podés subir tu repositorio para compartirlo con el equipo o tener una copia de respaldo en la nube.

En el ejercicio usaste Git localmente. Subir a GitHub es el paso siguiente — cuando estés lista para compartir.

💡 Los proyectos IT4W están en el repositorio de SharePoint. Git es el paso local; SharePoint es el paso compartido.
