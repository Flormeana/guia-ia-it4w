Ejercicio Final — Guía IA IT4W

Ejercicio de Inicio:
Hacelo vos, todo en orden

Tiempo: 20–30 minutos
Nivel: Cualquier área
Requiere: VS Code abierto
Este ejercicio pone en práctica todo lo que aprendiste en la guía. No necesitás saber programar. Solo seguí los pasos en orden. Al terminar, vas a haber usado los tres modos, dos subagentes y Git.
01
Modo Ask
Explorá el proyecto
Aprender a explorar sin tocar nada

Seleccioná el modo Ask en el panel de chat de Copilot y escribí exactamente esto:

¿Qué archivos tiene este proyecto?

La IA te lista los archivos y explica para qué sirve cada uno. No crea nada, no modifica nada.

Chequeá que lo entendiste:

  • ¿Dónde están los archivos de contenido?
  • ¿Dónde están los skills?
  • ¿Qué hace el archivo copilot-instructions.md?
02
Modo Plan
Planificá antes de hacer
Aprender a pensar antes de ejecutar

Seleccioná el modo Plan y escribí exactamente esto:

Planificá cómo escribirías la sección sobre agentes

La IA te devuelve un plan en pasos — todavía no escribe nada, te muestra cómo pensaría la tarea.

Podés responderle al plan: "Cambiá el ejemplo por uno de UX en lugar de comercial" y va a ajustar antes de ejecutar.
03
Modo Agent — Subagente
Ejecutá con un subagente
Invocar un subagente por primera vez

Seleccioná el modo Agent y escribí exactamente esto:

/generar-seccion concepto: agentes archivo: prueba-agentes.md

El subagente /generar-seccion redacta la sección y la guarda en prueba-agentes.md.

Verificá: ¿El archivo existe en el explorador? ¿Usa el tono IT4W?

04
Modo Agent — Revisión
Revisá con el subagente revisor
Entender el score de simplicidad

Seguí en modo Agent y escribí exactamente esto:

/revisar-simplicidad archivo: prueba-agentes.md

El subagente evalúa el texto y te da un score + feedback.

ScoreQué significa
1–4Texto muy técnico, hay que reescribir
5–6Entendible pero mejorable, revisar sugerencias
7–8Buen nivel, apto para la guía
9–10Excelente, cualquier persona en IT4W lo entiende
05
Terminal
Guardá tu trabajo con Git
Entender qué hace Git y por qué usamos commits

Git es como el historial de versiones de Google Docs, pero más potente. Cada commit guarda un punto al que podés volver si algo sale mal.

Abrí la terminal en VS Code (Menú Terminal → New Terminal) y ejecutá:

# Marcá todos los archivos nuevos o modificados git add . # Guardá el punto con un mensaje que explica qué hiciste git commit -m "ejercicio: primera sección generada"

Si ves algo como [main abc1234] ejercicio: primera sección generada, ¡Git guardó tu trabajo!

¡Lo lograste!
