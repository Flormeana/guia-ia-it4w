Sección 01

Los modos de trabajo:
Ask, Plan y Agent

VS Code es la oficina. La IA es el equipo que trabaja adentro.

Cuando abrís el chat de GitHub Copilot en VS Code, podés elegir cómo querés que la IA trabaje con vos. Hay tres modos, cada uno con un nivel distinto de autonomía.

Pensalo como pedirle algo a un asistente: Ask = "¿me explicás esto?"  ·  Plan = "¿cómo lo harías vos?"  ·  Agent = "hacelo vos, yo te sigo."
🔍
Ask
Para explorar y entender
Hacés una pregunta, la IA responde. No toca nada, no modifica nada. Solo te explica o te orienta.
Ejemplo IT4W — UX "¿Qué archivos tiene este proyecto y para qué sirve cada uno?"
🗺️
Plan
Para pensar antes de hacer
Le pedís a la IA que te proponga cómo abordar una tarea. Te da una hoja de ruta, no una acción.
Ejemplo IT4W — QA "Planificá cómo escribirías la documentación de este flujo de pruebas."
🤖
Agent
Para ejecutar y delegar
La IA no solo responde: toma decisiones, usa herramientas y ejecuta tareas por vos.
Ejemplo IT4W — Dashboards "/generar-seccion concepto: KPIs de ventas archivo: contenido/kpis.md"

¿Cómo se usa?

  1. Abrí el panel de chat de Copilot en VS Code
  2. Seleccioná el modo: Ask, Plan o Agent
  3. Escribí tu pedido en lenguaje natural — sin saber programar
  4. Leé la respuesta y seguí desde ahí
💡 Si no estás segura de cuál usar, empezá con Ask. Siempre podés cambiar de modo.

Tabla de decisión rápida

ModoLo que hace la IALo que vos hacés
AskResponde y explicaPreguntás y aprendés
PlanPropone un caminoDecidís si seguirlo
AgentEjecuta y produceSupervisás el resultado
Sección 02

GitHub Copilot vs Claude:
el mismo lugar, distinto motor

El mismo lugar, distinto motor.

En VS Code podés usar distintos modelos de IA dentro del mismo chat de Copilot. GitHub Copilot es el asistente de Microsoft. Claude es el modelo de Anthropic. Ambos viven en el mismo panel, pero piensan distinto.

Es como elegir entre dos médicos especialistas: los dos son buenos, pero uno tiene más experiencia en lo que necesitás.
🔷
GitHub Copilot
Microsoft · Modelo base
Especializado en código y completado de texto. Sugerencias mientras escribís. Integración nativa con VS Code.
🟣
Claude Sonnet
Anthropic · Modelo de lenguaje
Especializado en razonamiento y lenguaje natural. Entiende contexto largo y documentos. Ideal para redactar, revisar y coordinar flujos. Lee y aplica skills con precisión.

¿Cómo se cambia el modelo?

  1. Abrí el chat de Copilot en VS Code
  2. Hacé clic en el selector de modelo (arriba en el panel, muestra el nombre actual)
  3. Elegí: GPT-4o, Claude Sonnet, etc.
  4. Escribí tu pedido normalmente — el resto es igual

Ejemplo — Sofía, diseñadora UX

❌ Con Copilot base
Respuesta genérica, en inglés, pensada para código.
✅ Con Claude Sonnet
Lee el archivo de instrucciones del proyecto, entiende el tono IT4W y redacta directo en español con el estilo del equipo.

¿Cuándo uso cada uno en IT4W?

ModeloFuerte en...Cuándo usarlo
GitHub CopilotCódigo, sugerencias rápidasArchivos de configuración técnica
Claude SonnetLenguaje, contexto, flujosRedacción, revisión, orquestación
Sección 03

Skills: el manual de
comportamiento de la IA

Un skill es el manual de procedimientos que el especialista tiene sobre su escritorio.

Un skill es un archivo que le dice a la IA cómo comportarse en una tarea específica — el tono, el formato, los pasos y las reglas que tiene que seguir.

Pensalo como el manual de onboarding que le das a alguien nuevo en el equipo: "En IT4W hacemos las cosas así, con este tono, siguiendo estos pasos."

Skills disponibles en el proyecto

🎨
brand-guidelines
Diseño y visual
Identidad visual IT4W — colores, tipografías y estilo de marca.
🖼️
canvas-design
Diseño y visual
Diseño de layouts, posters y piezas visuales estáticas.
✍️
doc-coauthoring
Documentos y contenido
Redacción colaborativa de documentos, specs y propuestas.
📢
internal-comms
Documentos y contenido
Comunicación interna: newsletters, updates, reportes.
📊
pptx / pdf / docx
Exportación final
Exportar el contenido como presentación, PDF o documento Word.

¿Cómo se usa un skill?

Opción 1 — Copilot lo activa automáticamente: Lee las instrucciones del proyecto y sabe qué skill cargar según la tarea.

Opción 2 — Lo cargás vos:

#file:skills-main/skills/doc-coauthoring/SKILL.md
Redactá la introducción del informe mensual de UX.

Ejemplo: newsletter de Comercial

❌ Sin skill
La IA produce texto genérico, formal y en inglés. No sigue el formato del equipo.
✅ Con internal-comms
La IA sigue el formato de comunicación interna IT4W: progreso, planes y problemas — en español, listo para copiar.
⚡ Regla importante: Cada sesión empieza desde cero. Los skills son la forma de que la IA recuerde las reglas del equipo.
Sección 04

Agentes y subagentes:
el director y los especialistas

Un agente = un archivo .md con instrucciones. No importa cómo lo creaste, siempre termina siendo ese archivo.

Un agente es una IA que puede tomar decisiones, usar herramientas y completar tareas por sí sola. Un subagente es otro agente que el primero puede "llamar" para una tarea específica.

Como una producción audiovisual: el director coordina, y llama al especialista en sonido, al de iluminación y al de edición. Cada uno hace su parte. El director ensambla todo.

El sistema de agentes en IT4W

🤖 Agente Principal · Copilot + Claude en modo Agent
↓ delega según la tarea
/generar-seccion
Redacta una sección de la guía con el formato y tono IT4W
/revisar-simplicidad
Evalúa si el texto es claro para personas no técnicas (score 1-10)
/diseñar-seccion
Genera el diseño visual de la sección aprobada

¿Cómo se invoca?

En modo Agent, escribís simplemente:

/generar-seccion concepto: agentes y subagentes archivo: contenido/04-agentes.md

Flujo real — Comercial pide una sección nueva

  1. Escribe en modo Agent: "Generá la sección sobre dashboards para analytics"
  2. El agente principal recibe el pedido y lee las reglas del proyecto
  3. Invoca /generar-seccion con el concepto y el archivo destino
  4. El subagente redacta el contenido con el estilo IT4W
  5. El agente invoca /revisar-simplicidad automáticamente
  6. Si score ≥ 7, invoca /diseñar-seccion
  7. Resultado final en contenido/ y diseño/
💡 Todo esto pasa con un solo pedido inicial. La IA trabaja como un equipo, no como una sola persona.
Sección 05

MCP: el puente entre
la IA y otras herramientas

MCP es el teléfono que le permite a la IA hablar con herramientas fuera de VS Code.

MCP (Model Context Protocol) es la forma en que la IA se conecta con otras herramientas y servicios externos. Le da a la IA "manos" para trabajar con el mundo fuera del chat.

Es como darle a un asistente acceso a tus aplicaciones: ahora no solo puede decirte cómo hacer algo, sino hacerlo directamente en Figma, Notion, Slack o donde sea.

Sección 06

Orquestación: un pedido,
todo el equipo trabajando

El agente principal es el director de orquesta: vos le decís qué querés lograr, y él decide qué especialista llamar y en qué orden.

Orquestación es cuando la IA coordina automáticamente múltiples pasos y agentes para completar una tarea compleja. Vos hacés un solo pedido. La IA arma el equipo y entrega el resultado.

Como llamar a un catering para un evento: vos decís "quiero un asado para 30 personas el sábado" y ellos organizan todo. Vos no coordinás a cada persona — solo hacés el pedido.

Sin orquestación vs Con orquestación

❌ Sin orquestación — 6 pasos manuales
1. Pedirle que redacte
2. Copiar el resultado
3. Pedirle que lo revise
4. Copiar de nuevo
5. Pedirle que lo diseñe
6. Guardarlo en el lugar correcto
✅ Con orquestación — 1 pedido
Escribís el pedido inicial.
El flujo corre solo.
Los pasos se encadenan.
Los archivos se guardan.
El diseño se genera.
Revisás el resultado final.

El flujo de orquestación en IT4W

↓

🔒 Si score < 7 → vuelve a /generar-seccion para reescribir
↓ Si score ≥ 7
🎨
/diseñar-seccion Genera el diseño visual de la sección

Ejercicio Final

Ejercicio de inicio:
Hacelo vos, todo en orden

Git no guarda en otro lado — guarda la historia dentro de tu carpeta.

Este ejercicio pone en práctica todo lo que aprendiste. No necesitás saber programar. Solo seguí los pasos en orden. Al terminar vas a haber usado los tres modos, dos subagentes y Git.

⏱️
20–30 min
Tiempo estimado
👩‍💻
Cualquier área
No se requiere experiencia técnica
🗂️
VS Code abierto
Con este proyecto cargado
01
Modo Ask
Explorá el proyecto
Aprender a explorar sin tocar nada

Seleccioná el modo Ask y escribí:

¿Qué archivos tiene este proyecto?

La IA te lista y explica. No crea nada, no modifica nada.

02
Modo Plan
Planificá antes de hacer
Aprender a pensar antes de ejecutar

Seleccioná el modo Plan y escribí:

Planificá cómo escribirías la sección sobre agentes

La IA te devuelve un plan en pasos — todavía no escribe nada.

Podés responderle: "Cambiá el ejemplo por uno de UX" y va a ajustar antes de ejecutar.
03
Modo Agent — Subagente
Ejecutá con un subagente
Invocar un subagente por primera vez

Seleccioná el modo Agent y escribí:

/generar-seccion concepto: agentes archivo: prueba-agentes.md

El subagente redacta la sección y la guarda en el archivo indicado.

04
Modo Agent — Revisión
Revisá con el subagente revisor
Entender el score de simplicidad

Seguí en modo Agent y escribí:

/revisar-simplicidad archivo: prueba-agentes.md
1–4Texto muy técnico, hay que reescribir
5–6Entendible pero mejorable, revisar sugerencias
7–8Buen nivel, apto para la guía
9–10Excelente, cualquier persona en IT4W lo entiende
05
Terminal
Guardá tu trabajo con Git
Entender qué hace Git y por qué usamos commits

Git es como el historial de versiones de Google Docs, pero más potente. Cada commit guarda un punto al que podés volver si algo sale mal.

Abrí la terminal (Ctrl + ñ) y ejecutá:

git add .
git commit -m "ejercicio: primera sección generada"

Si ves [main abc1234] ejercicio: primera sección generada, ¡Git guardó tu trabajo!
